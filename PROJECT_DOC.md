# 中国近五年天气抓取与可视化分析项目文档

## 1. 项目概述

本项目是一个面向展示和分析场景的中国天气可视化项目，目标是完成以下工作：

1. 抓取中国主要城市近五年的历史天气数据。
2. 对数据进行清洗、补全、聚合与趋势分析。
3. 输出交互式可视化大屏。
4. 输出分析摘要报告。
5. 支持地图时间轴、指标切换、省份联动趋势等交互能力。

当前主脚本：

- [china_weather_spider_analysis.py](weatherVisualization/china_weather_spider_analysis.py)

当前主要输出：

- [china_weather_dashboard_5y.html](weatherVisualization/outputs/figures/china_weather_dashboard_5y.html)
- [china_weather_report_5y.md](weatherVisualization/outputs/reports/china_weather_report_5y.md)
- [china_weather_daily_5y.csv](weatherVisualization/data/processed/china_weather_daily_5y.csv)
- [china_weather_monthly_5y.csv](weatherVisualization/data/processed/china_weather_monthly_5y.csv)

## 5. 核心功能

### 5.1 数据抓取

项目按“运行当天向前回溯 5 年”计算抓取区间。

例如在 `2026-04-16` 运行时：

- 开始日期：`2021-04-17`
- 结束日期：`2026-04-16`

抓取逻辑特点：

- 支持多城市抓取
- 支持缓存复用
- 支持 `--force-refresh` 强制重新抓取
- 支持历史缓存回退
- 支持按行政区划动态生成地级市抓取目标

### 5.2 历史缓存回退

这是后期非常重要的一次增强。

因为用户环境里曾多次出现：

- API 限流 `429`
- 代理连接失败
- 新日期导致缓存文件名变化

为避免“当天抓不到就把总表覆盖成残缺版本”，脚本增加了：

- 如果新日期对应缓存不存在，优先查找该城市最近一次成功缓存
- 如果请求失败，自动回退到历史缓存

这个机制解决了“很多城市突然没有数据”的问题。

补充：

- 为避免地级市扩展后出现重名城市缓存互相覆盖，当前缓存命名已优先使用行政区代码。

### 5.3 数据清洗与聚合

清洗内容包括：

- 字段标准化重命名
- 日期拆分为年、月、年-月
- 按城市做数值字段插值补全
- 生成温差、舒适度等衍生指标

聚合输出包括：

- 日度明细表
- 月度聚合表
- 城市升温趋势表

### 5.4 文本报告

报告输出：

- 全国年度平均气温
- 近五年均温最高城市
- 降水较强城市
- 升温趋势明显城市
- 舒适度较高城市

### 5.5 大屏数据控制

当前大屏已新增本地数据控制能力：
- 支持“更新全部数据”
- 支持“仅重抓失败数据”
- 支持“中断任务”
- 支持显示总进度、成功数、失败数和当前处理城市
- 支持将失败城市写入 `.api_cache/failed_cities.json`
- 支持通过 `start_dashboard.bat` 一键启动本地控制服务并打开大屏
- `--serve-dashboard` 当前默认会优先快速启动本地服务，不再先阻塞执行整套全量抓取
- 如需启动服务前先跑一次全量更新，可额外使用 `--init-update`
- “仅重抓失败数据”当前直接使用失败清单，不再依赖重新匹配动态城市列表
- 抓取核心逻辑已补充中文注释，便于后续维护和继续迭代

## 6. 可视化大屏说明

当前大屏为一个固定栅格布局的 HTML 仪表盘，不是默认拖拽页。

### 6.1 顶部摘要卡片

用于展示：

- 当前样本覆盖城市数
- 当前覆盖省级区域数
- 当前时间序列最新月份
- 当前待重抓失败城市数

### 6.2 中国分省气候时序地图

功能：

- 时间轴拖动查看不同月份
- 切换指标：
  - 平均气温
  - 降水量
  - 平均风速
- 自动播放
- 点击省份后联动下方趋势图
- 双击省份后进入该省城市视图

地图实现说明：

- 省级数据来自城市聚合后的 `province + year_month`
- 使用省名映射解决 `北京 -> 北京市`、`新疆 -> 新疆维吾尔自治区` 这类名称不一致问题
- 地图底图通过在线 GeoJSON 加载
- 当前自动播放已改为直接切换月份，不再做插值过渡
- 省内视图当前使用城市散点显示该省城市天气指标

### 6.3 省份联动月度趋势图

功能：

- 默认显示全国平均月度趋势
- 点击地图省份后切换到该省趋势
- 点击趋势图月份点可反向切换地图时间

说明：

- 按用户要求，趋势图中的“当前月份高亮线”和“高亮点”已移除
- 当前仅保留趋势图本体与地图的时间联动
- 双击进入省内地图后，趋势图仍继续按当前选中省份联动

### 6.4 其他图表

当前大屏还包括：

- 中国主要城市近五年气候空间分布
- 当前样本月度温度趋势
- 区域月度平均气温热力图
- 升温趋势城市排行
- 典型区域气候雷达图
- 温度与降水关系散点图

补充说明：
- 除地图外的摘要卡片和图表说明已统一改为“当前样本”口径
- 非地图板块当前反映的是已成功落地样本，不再默认写成完整全国结论
- 地图区域已增加数据控制按钮，可直接触发全量更新、失败重抓和中断任务
- 数据任务运行中会临时禁用两个更新按钮，避免重复点击启动多个任务
- 任务完成后会弹出提示，并自动刷新页面状态
- 当前任务状态信息也会在地图区域展示，便于观察数据更新进度
- 中断按钮现在会即时反馈“已发送中断请求”，并且等待与重试阶段也能响应中断
- 地图上方按钮布局已重新分组，按“指标/播放控制”和“数据任务控制”两层排列，更规整

## 7. 运行方式

### 7.1 安装依赖

```powershell
pip install -r requirements.txt
```

### 7.2 常规运行

```powershell
python china_weather_spider_analysis.py
```

### 7.3 强制重抓

```powershell
python china_weather_spider_analysis.py --force-refresh
```

### 7.4 限制并发

```powershell
python china_weather_spider_analysis.py --workers 1
```

说明：

- 当前公共接口存在限流与代理问题，建议优先使用 `--workers 1`
- 这是目前最稳的运行方式

### 7.5 启动大屏本地控制服务

```powershell
python china_weather_spider_analysis.py --serve-dashboard --workers 1
```

说明：
- 大屏中的数据更新按钮依赖本地 HTTP 控制服务
- 如果没有先启动该服务，按钮无法直接触发 Python 抓取任务
- 启动后可通过 `http://127.0.0.1:8765/dashboard` 打开可交互版本
- 当前默认会优先进入本地服务，适合先开大屏再按需点击页面按钮更新数据

### 7.5.1 启动服务前先执行一次全量更新

```powershell
python china_weather_spider_analysis.py --serve-dashboard --workers 1 --init-update
```

### 7.6 一键启动

可直接双击：
- [start_dashboard.bat](weatherVisualization/start_dashboard.bat)

作用：
- 自动启动本地控制服务
- 自动打开大屏地址

## 9. 版本迭代记录

这一部分按本次开发过程整理，重点记录功能演进和修复点。

### V1：离线示例数据版

最开始版本不是抓真实数据，而是：

- 内置中国主要城市示例气候数据
- 输出多图表 HTML 大屏

特点：

- 可快速展示效果
- 不依赖网络

问题：

- 不是实际爬取
- 不符合“抓取近五年中国天气数据”的需求

### V2：Meteostat 历史天气抓取版

后续改为：

- 使用 `Meteostat` 抓取近五年中国城市历史天气

优点：

- 接口结构清晰
- 能直接拉历史数据

遇到的问题：

1. Python 3.14 环境下依赖兼容麻烦
2. `meteostat` 顶层 API 与预期不同
3. 本地缓存目录并发创建报错
4. `data.meteostat.net` 网络连接在用户环境中被拒绝

结论：

- 最终放弃作为主数据源

### V3：切换到 Open-Meteo Historical Weather API

为了绕开 `Meteostat` 的连接问题，改为：

- 使用 `Open-Meteo Historical Weather API`

这一版解决了：

- 历史天气可按经纬度抓取
- 适合按城市点位做近五年分析

但随后遇到：

- 公共接口 `429 Too Many Requests`

### V4：限流与重试机制

为解决限流，加入：

- 降低并发线程数
- 对 `429` 做重试与退避
- 默认推荐 `--workers 1`

效果：

- 抓取稳定性提升

### V5：项目目录结构整理

输出目录从扁平结构整理为：

- `data/raw`
- `data/processed`
- `outputs/figures`
- `outputs/reports`

目标：

- 便于管理原始数据、结果数据、图表和报告

### V6：高级大屏布局重构

大屏从默认拼接图页升级为固定布局仪表盘。

改动：

- 增加顶部摘要卡片
- 使用统一卡片式面板
- 统一背景、字体、颜色和阴影
- 调整图表尺寸和栅格布局

效果：

- 页面更规整
- 更像成品展示页

### V7：时间轴地图加入

加入分省气候地图面板，支持：

- 时间轴
- 指标切换
- 自动播放

这是项目从“静态分析图集合”升级成“交互式大屏”的关键版本。

### V8：地图与趋势联动

加入：

- 点击地图省份，切换下方趋势图
- 趋势图点击月份，反向控制地图时间轴

这一步让地图和趋势图真正变成双向联动。

### V9：地图动画、过渡和插值

这一阶段持续迭代了很多次：

- 加播放节奏控制
- 加月份进度展示
- 加更平滑的动画缓动
- 月度切换做插值过渡
- 去掉整图淡入淡出闪烁

最终保留：

- 插值播放
- 缓动过渡

最终移除：

- 全图闪烁式淡入淡出

原因：

- 用户明确反馈“闪烁很难受”

### V10：地图颜色体系迭代

地图颜色经历了多轮调试：

1. 单色蓝阶
2. 更细的连续蓝阶
3. 高对比低蓝高红
4. 保留低蓝高红，但做连续渐变

最终目的：

- 颜色差异明显
- 月份切换时仍然平滑

### V11：月份色带动态范围

用户指出地图下方的色带数值不准确。

根因：

- 色带使用的是五年全局最小/最大值
- 当前月份的数据范围与其不匹配

修复：

- 改成按当前月份动态更新 `visualMap` 的 `min/max`
- 插值过渡时 `min/max` 也平滑过渡

### V12：缓存回退机制

用户环境在后期出现新的问题：

- 日期改变导致缓存名变化
- 代理拒绝连接 API
- 结果表只剩部分城市

修复：

- 新日期抓取失败时自动回退到最近一次历史缓存
- 避免覆盖成残缺版本

这一步非常关键，显著增强了项目可用性。

### V13：台湾省支持

用户提出台湾省没有数据。

改动：

- 加入 `台北`
- 加入 `台中`
- 加入 `高雄`
- 增加 `台湾 -> 台湾省` 地图映射

当前状态：

- 代码已支持台湾
- 但用户当前代理环境无法成功抓取台湾数据
- 因此地图上台湾可能仍为空

### V14：趋势图当前月份高亮反复迭代后移除

这部分是整个项目中最反复的一段迭代。

先后尝试过：

1. `markLine`
2. 外部 `graphic` 自画竖线和点
3. 图形层在 `finished` 后重画
4. 第二高亮系列 + `axisPointer`
5. 第三系列竖线

原因：

- 用户环境中该高亮线始终存在“第一次动，后面不动”的问题
- 同时一度引入了前端脚本错误，导致地图和趋势图都不显示

最终决策：

- 彻底移除趋势图高亮线和高亮点
- 保留趋势图本体与地图联动

这是一个典型的工程取舍：

- 放弃低收益但高不稳定性的细节效果
- 保住主功能稳定

### V15：地级市抓取与省内钻取升级

本次版本新增：

- 抓取目标从固定样本城市扩展到按行政区划动态生成地级市列表

### V16：大屏数据控制与任务中断

### V17：抓取容错与大屏非地图板块整理
本次版本新增：

- 中断按钮修正了任务启动与取消之间的竞态，并增强了等待阶段的中断响应
- 大屏除地图外的摘要卡片与图表说明改为“当前样本”口径，更贴合实际已落地数据
- 核心抓取与控制逻辑补充中文注释，方便后续维护

## 10. 当前已知问题

### 10.1 台湾数据未实际落地

原因：

- 当前用户网络/代理环境拒绝连接 `open-meteo`
- 台湾新增城市没有历史成功缓存可回退

结果：

- 台湾城市已在代码中
- 但 processed 数据里可能仍无台湾

### 10.2 地图依赖在线底图

地图 GeoJSON 当前为在线加载。

结果：

- 如果浏览器网络不通，地图底图可能不显示

后续优化方向：

- 将中国地图 GeoJSON 文件放入项目本地
- 不再依赖在线底图请求

### 10.3 API 依赖公共服务

风险：

- 429 限流
- 代理冲突
- 单日运行时新日期触发新缓存名

当前缓解措施：

- 建议 `--workers 1`
- 历史缓存回退

### 10.4 地级市结果是否落地取决于网络环境

当前代码已经支持地级市抓取，但如果运行环境网络或代理异常：

- 行政区划接口可能无法访问
- 天气接口可能无法访问
- 程序只能继续使用旧缓存或旧样本城市数据

这会导致省内地图虽然已经支持钻取，但页面中实际显示的城市数量仍可能很少。

### 10.5 大屏数据控制依赖本地服务

当前大屏中的数据更新按钮不是纯静态 HTML 能独立完成的，仍依赖本地 Python 控制服务。
因此如果没有先启动：
- `python china_weather_spider_analysis.py --serve-dashboard --workers 1`
或：
- [start_dashboard.bat](weatherVisualization/start_dashboard.bat)

则 `http://127.0.0.1:8765/dashboard` 无法访问，按钮也无法实际触发数据抓取任务。

### 10.6 非地图板块仍受样本覆盖范围限制

当前大屏除地图外的摘要卡片、趋势图、热力图、雷达图和散点图，已经统一调整为“当前样本”口径。
这能避免把局部成功样本误写成完整全国结果，但也意味着：
- 当 `processed` 结果仍不完整时，非地图板块反映的是当前已落地样本，而不是全国完整情况
- 如果失败城市较多，板块统计结论会随补抓进度继续变化

## 11. 后续可继续优化的方向

### 11.1 离线化地图底图

建议：

- 将中国 GeoJSON 文件放入项目本地
- 不再依赖在线底图请求

### 11.2 台湾数据离线兜底

建议：

- 单独准备台湾 3 城市本地缓存
- 即使网络失败，台湾也能显示

### 11.3 顶部摘要卡片动态联动

建议：

- 随当前省份和当前月份联动更新

### 11.4 更正式的项目交付版本

建议：

- 增加封面页
- 增加分析结论页
- 增加极端天气异常说明

## 12. 总结

这个项目已经从最初的静态示例版，演进为一个具备以下能力的可运行天气分析系统：

- 近五年中国城市天气抓取
- 缓存复用与网络失败回退
- 日度与月度数据输出
- 交互式地图时间轴
- 地图与趋势图双向联动
- 成品化大屏布局

同时，这个项目也保留了完整的工程迭代痕迹：

- 数据源替换
- 依赖兼容问题处理
- 地图颜色与动画多轮调优
- 交互联动设计反复修正
- 对不稳定功能进行降级取舍

如果要继续迭代，最值得优先做的是：

1. 本地化地图底图
2. 为台湾补充离线缓存
3. 增加顶部卡片联动与异常天气分析
